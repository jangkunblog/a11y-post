# 장군블로그 글쓰기 패턴 (참고 기준)

기준 글: https://jangkunblog.com/wp/skip-navigation-is-not-a-quick-link/

## 구조 패턴

- 오해 바로잡기형 제목을 사용한다.
  - 예: "X는 Y가 아닙니다"
- 초반부에서 개념을 명확히 정의한다.
- 국내외 지침/표준을 날짜와 함께 제시한다.
- "적절한 사용방법"과 "잘못된 사례"를 분리해 설명한다.
- UI 관점과 개발 검수 방법을 별도 섹션으로 다룬다.
- 마지막에 정리(마무리)와 외부 참고 링크를 제공한다.

## 문단/톤 패턴

- 한 문단은 보통 2~4문장으로 끊는다.
- 핵심 주장 문장은 단정형으로 명확히 쓴다.
  - 예: "건너뛰기 링크는 문서의 처음에 한 번만 제공하는 것이 적절합니다."
- 설명은 원인 -> 영향 -> 권장안을 순서대로 전개한다.
- 과장 없이 실무적이고 교육적인 톤을 유지한다.
- 산문형을 기본으로 쓰되, 절차/비교/요약/검수 항목은 리스트로 전개해 이해 비용을 낮춘다.

## 표현 패턴

- 중요한 결론은 문장 자체를 반복해 강조한다.
  - 예: "다시 말씀드리지만..."
- 용어는 한글 우선 + 괄호로 영문 병기한다.
- 독자가 바로 적용할 수 있도록 코드/검수 절차를 포함한다.

## 분량 가이드 (기본 목표)

- H2: 8개 이상
- H3: 8개 이상
- 본문 단락: 30개 이상
- 코드 예시: 1개 이상
- 검수 체크리스트: 1개 이상

## 적용 방법

1. 초안 생성 시 위 구조를 먼저 잡는다.
2. 각 섹션에서 "정의 -> 근거 -> 사례 -> 행동" 순서를 유지한다.
3. 결론에서 핵심 행동 지침을 3줄 이내로 재정리한다.
